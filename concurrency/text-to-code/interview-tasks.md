## 1) (Без компании, базовая задача) Напиши worker pool. 
- ### Есть слайс tasks []string
- ### Максимум 3 горутины работают одновременно
- ### Каждая горутина вызывает process(task string) 
абстрактная функция , просто `func process(task string) {}` без логики
- ### Если контекст отменён — все воркеры останавливаются
- ### Дожидаемся завершения всех горутин


## 2) (Компания iTrend)  Напиши Fan out , на собесе это не обговаривали, здесь скорее уточнее 
- ### Есть urls []string
- ### Каждый URL обрабатывается в отдельной горутине
- ### Если горутина получила сетевую ошибку (net.Error) — останавливаем всех остальных
- ### Результат запроса не важен, просто "прозваниваем"


## 3) (Компания Rusiem)  Напиши `Retry(ctx context.Context, maxAttempts int, fn func() error) error`. 
- ### Функция повторяет `fn` до успеха или исчерпания попыток. Между попытками - экспоненциальная задержка (1s, 2s, 4s...).
- ### Функция должна поддерживать отмену через `ctx`.  
    

## 4) (Компания NDA из сферы гемблинга) Напиши функцию `merge` которая принимает произвольное количество каналов и возвращает один канал в который пишутся все значения из входных каналов.
- ### Каналы обрабатываются параллельно — не ждём пока закроется первый чтобы начать читать второй
- ### Результирующий канал закрывается когда все входные каналы закрыты
- ### Порядок значений не важен
- ### Усложения для второго этапа: Добавь ограничение по времени — функция должна работать не дольше одной секунды. Если входной канал завис и не закрывается — всё равно завершаемся через секунду. (Тут есть вариант как с таймером, так и с контекстом, если есть желание, можно реализовать сразу оба варианта)
